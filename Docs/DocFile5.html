        Step 4: Create a front end

        In this step you are going to create a user interface. This should allow users to check-in and -out parking spaces.

        Prerequisites

        1. Visual Studio Code
          • If you're not already using VS Code, now it's time to install it.
          • Install all the following extensions:
            • Angular Language Service
            • Debugger for Chrome
            • TSLint
        2. Install Node.js version manager for Windows
          • Angular is a framework based on Node.js. Install the NVM to manage different versions of Node on a single machine.
          • Follow the guide from the Github repository coreybutler/nvm-windows
        3. Install Node.js LTS version
          • Install the latest version of Node.js using the nvm command util. Make sure to chose a LTS version to make sure it is stable.
        4. Install the Angular CLI
          • Using the Node package manager (npm), you can easily install the Angular CLI. It contains all the required commands for creating, building and debugging your Angular app.
            npm install -g @angular/cli

        Tasks

        Structure

        Create a new Angular workspace inside your GIT repository. Similar to the dotnet CLI, the Angular CLI helps you to do so. Using the Angular CLI, create the following structure of the project:

        Routing for users and for administrators

        Create two components, one called parking-user and the other called parking-manager. In the AppRoutingModule, define the following two routes:

        • "/": Default route. When opening the frontend, theparking-user component should be loaded by default. Access to this route has anyone.
        • "/admin/": Administrator section. When navigating to this route, the parking-manager route should be displayed. This section will be protected in the next step.
        • For all the other routes, as they don't exist, a page indicating the "error 404 not found" should be displayed.

        Admin guard on route "/admin/"

        We now want to restrict access to the administration view for non-admin users. Implement a guard that blocks the access for all users, that don't have an admin token in the app's local storage. Since we don't implement any authentication in this project,you can test it by manually adding the key admin_token to your local storage (F12 in your browser -> Application -> Local Storage -> http://localhost:4200):

        If this key isn't present on the browser, the access must be restricted.

        Implement component for user check-in and checkout

        Now your creativity is requested! Implement a page where parking users can check-in and check-out, using our API service. Make sure to seperate the API interaction (from the rest of the display logic) into an ownclass, a so-called service. Use the Angular CLI to generate such a class.

        Implement component for managing parking spaces

        Now implement a page for the parking manager. As a parking manager, I want to see a list of all the parking spaces with their occupation status. I want to be able to create a new parking space or delete an existing one, to easily extend or shrinkthe parking space capacity.

        Acceptance criteria

        • The frontend contribution guide and definition of done arefollowed
        • The application follows the structural guide.
        • The admin section is protected by a guard that checks the "admin_token"
        • The user section offers check-in and check-out functionality
        • The admin section offers list, create and deletion of parking spaces
        • API interaction is seperated in a service and not inside a component.
        • Error codes are handled and the user is notified with what he needs to change to successfully execute the action he wanted (e.g. Choose another space if it is occupied).
